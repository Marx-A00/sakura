  <action>
    **1. Create `LocalStorageBackend.kt`** in `com/sakura/sync/`:
    - Implements `SyncBackend` interface
    - Constructor receives `Context` and optional `CoroutineDispatcher` (default `Dispatchers.IO`)
    - `resolveFile(filename)` returns `File(context.filesDir, filename)` — no pref lookup needed
    - `readFile`: no permission check, reads from `context.filesDir`, returns empty string if file doesn't exist
    - `writeFile`: uses `AtomicFile` for crash-safe writes (identical pattern to `SyncthingFileBackend`)
    - `fileExists`: simple `resolveFile(filename).exists()`
    - `listOrgFiles`: lists `context.filesDir` files filtered to `.org` extension, excluding dotfiles
    - `checkSyncStatus`: returns `SyncStatus(lastSyncedAt = null, hasConflicts = false, folderAccessible = true)` — always accessible, no sync concept
    - NO `checkPermission()` method — internal storage never requires permissions
    - NO import of `Environment` — not needed

    **2. Add `StorageMode` to `AppPreferencesRepository.kt`:**
    - Add `enum class StorageMode { LOCAL, SYNCTHING }` at the top of the file (outside the class, same file)
    - Add companion key: `val STORAGE_MODE = stringPreferencesKey("storage_mode")`
    - Add flow: `val storageMode: Flow<StorageMode?>` that maps the string pref to the enum using `StorageMode.valueOf()` wrapped in `runCatching { }.getOrNull()`. Returns `null` when no value stored (not yet selected).
    - Add setter: `suspend fun setStorageMode(mode: StorageMode)` that stores `mode.name`
    - Follow the existing error-catch pattern: `.catch { if (it is IOException) emit(emptyPreferences()) else throw it }`

    **3. Refactor `AppContainer.kt`:**
    - Add `storageMode: StorageMode` as a second constructor parameter: `class AppContainer(context: Context, storageMode: StorageMode)`
    - Change `syncBackend` initialization to: `when (storageMode) { StorageMode.LOCAL -> LocalStorageBackend(context); StorageMode.SYNCTHING -> SyncthingFileBackend(prefsRepo) }`
    - Add import for `LocalStorageBackend` and `StorageMode`
    - Everything else stays the same — `prefsRepo`, `orgParser`, `orgWriter`, `foodRepository`, `workoutRepository` are unchanged

    **4. Refactor `SakuraApplication.kt`:**
    - Replace `lateinit var container: AppContainer` with `lateinit var prefsRepo: AppPreferencesRepository`
    - In `onCreate()`: `prefsRepo = AppPreferencesRepository(this)` — no longer creates full `AppContainer`
    - This is because `AppContainer` now needs `StorageMode`, which comes from DataStore (async)

    **5. Refactor `MainActivity.kt`:**
    - Remove the old `val container = (application as SakuraApplication).container` line — this will cause a build failure since `SakuraApplication.container` no longer exists after step 4
    - Get `prefsRepo` from `SakuraApplication` instead: `val prefsRepo = (application as SakuraApplication).prefsRepo`
    - Collect `storageMode` alongside `onboardingComplete`:
      ```
      val storageMode by prefsRepo.storageMode.collectAsStateWithLifecycle(initialValue = null)
      ```
    - Gate rendering on BOTH values being non-null: `if (onboardingDone != null && storageMode != null)`
    - BUT handle first-launch case: if `onboardingDone == false` (new user), don't require `storageMode` — they haven't selected it yet. Use: `if (onboardingDone != null) { ... }` and resolve mode inside:
      ```
      val resolvedMode = storageMode ?: StorageMode.LOCAL
      val container = remember(resolvedMode) { AppContainer(applicationContext, resolvedMode) }
      ```
    - `remember(resolvedMode)` ensures container re-creates when mode changes (after migration)
    - `startWithOnboarding = onboardingDone != true` logic stays the same
    - Import `StorageMode`, `AppPreferencesRepository`, `remember`
  </action>
