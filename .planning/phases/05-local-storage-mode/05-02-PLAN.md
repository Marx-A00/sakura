  <action>
    **1. Add Storage section to `MacroTargetsScreen.kt`:**
    - Below the existing macro target form, add a divider and a "Storage" section header
    - Read `prefsRepo.storageMode` flow and display the current mode:
      - `StorageMode.LOCAL` -> "Just me (local storage)"
      - `StorageMode.SYNCTHING` -> "Sync across devices"
      - `null` -> "Not configured" (shouldn't happen post-onboarding)
    - Add a "Change" button that triggers a migration confirmation dialog:
      - Show an `AlertDialog` with the target mode name and a warning about what happens:
        - Local -> Syncthing: "To switch to Sync mode, you'll need to grant storage access. This will restart the app."
        - Syncthing -> Local: "Your org files will be copied to local storage on this device."
      - "Cancel" dismisses; "Continue" triggers migration

    **2. Implement migration logic as a suspend function:**
    - `suspend fun migrateStorage(context: Context, prefsRepo: AppPreferencesRepository, targetMode: StorageMode)`
    - **Syncthing -> Local:**
      - Get the sync folder path from `prefsRepo.syncFolderPath.first()`
      - List all `.org` files in the sync folder
      - Copy each to `context.filesDir`
      - Call `prefsRepo.setStorageMode(StorageMode.LOCAL)`
      - Call `(context as ComponentActivity).recreate()` to force full restart with new AppContainer
    - **Local -> Syncthing:**
      - Call `prefsRepo.setStorageMode(StorageMode.SYNCTHING)`
      - Clear onboarding flag: `prefsRepo.clearOnboardingComplete()` (add this method if not present — just remove the key from DataStore)
      - Call `(context as ComponentActivity).recreate()` — this forces the user through the onboarding flow
      - OnboardingViewModel will detect stored SYNCTHING mode and skip Welcome/ModeSelection, going directly to permission/folder screens (see step 4)
      - Org file copying happens AFTER the user selects a sync folder in onboarding — the existing `onFolderSelected()` flow handles this naturally since the files already exist in `filesDir`

    **3. Wire Settings screen with context for migration:**
    - `MacroTargetsScreen` already receives `prefsRepo`. Use `LocalContext.current` for file operations and `(context as? ComponentActivity)?.recreate()` for restart.

    **4. Update `OnboardingViewModel.kt` for re-onboarding:**
    - Add an init block that checks if storageMode is already set:
      ```
      init {
          viewModelScope.launch {
              val existingMode = prefsRepo.storageMode.first()
              if (existingMode == StorageMode.SYNCTHING) {
                  // Re-onboarding for Syncthing setup — skip Welcome/ModeSelection
                  checkPermission()
              }
              // else: stay on Welcome (new user)
          }
      }
      ```
    - This handles the case where a local-mode user switches to Syncthing via Settings — the app restarts, onboarding shows, but the init block detects SYNCTHING mode and jumps straight to the permission check flow
    - Import `StorageMode` and `kotlinx.coroutines.flow.first`

    Note: Migration file-copy correctness (that the right files end up in the right place) cannot be verified at compile time. This is covered by the checkpoint:human-verify task (Test 3 — Mode switching from Settings).
