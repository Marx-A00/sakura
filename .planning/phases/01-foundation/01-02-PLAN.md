    6. `roundTrip_workoutFile` -- Same as above for workout data.

    7. `parseEmptyFile_returnsEmptyOrgFile` -- Parsing an empty string returns `OrgFile(sections = emptyList())`.

    8. `parseMalformedLine_skipsGracefully` -- A line that doesn't match any pattern is silently skipped. No crash.

    Run tests -- they must all FAIL.

    **GREEN -- Implement OrgParser:**

    Create `OrgParser.kt`:

    ```kotlin
    object OrgParser {
        /**
         * Parse org-mode text into an OrgFile model.
         * @param content Raw org file text
         * @param mode FOOD or WORKOUT -- determines how to parse entries under date sections
         */
        fun parse(content: String, mode: ParseMode): OrgFile

        enum class ParseMode { FOOD, WORKOUT }
    }
    ```

    Implementation approach:
    - Split content by lines
    - Walk lines sequentially, using a state machine:
      - When a line matches `OrgSchema.DATE_HEADING_REGEX` -> start new `OrgDateSection`
      - When a line matches `OrgSchema.MEAL_HEADING_REGEX` (in FOOD mode) -> start new `OrgMealGroup`
      - When a line matches `OrgSchema.FOOD_ENTRY_REGEX` (in FOOD mode) -> add entry to current meal group
      - When a line matches `OrgSchema.EXERCISE_ENTRY_REGEX` (in WORKOUT mode) -> add to current section's exercises
      - When a line matches a `** Heading` pattern in WORKOUT mode -> store as the workout label (for future use), or skip it
      - Blank lines and unrecognized lines are skipped gracefully
    - Use `OrgSchema.DATE_PARSE_FORMATTER` to parse the date from regex capture group
    - Parse macro integers with `.toInt()`, weight with `.toDouble()`
    - Return `OrgFile(sections)` when all lines are consumed

    Run tests -- they must all PASS.

    **REFACTOR:** If the state machine has deeply nested when/if blocks, extract helper functions for readability. Run tests again.
  </action>
