    **`SyncBackend.kt`** (interface):
    ```kotlin
    interface SyncBackend {
        suspend fun readFile(filename: String): String
        suspend fun writeFile(filename: String, content: String)
        suspend fun fileExists(filename: String): Boolean
        suspend fun listOrgFiles(): List<String>
    }
    ```

    **`SyncBackendError.kt`** (sealed class):
    ```kotlin
    sealed class SyncBackendError(message: String) : Exception(message) {
        class FolderUnavailable(message: String) : SyncBackendError(message)
        class PermissionDenied(message: String) : SyncBackendError(message)
        class ConflictDetected(val filename: String) : SyncBackendError("Conflict file detected: $filename")
    }
    ```

    **`SyncthingFileBackend.kt`:**
    - Constructor takes `AppPreferencesRepository` and `CoroutineDispatcher` (defaulting to `Dispatchers.IO`)
    - `resolveFile(filename)`: reads `prefsRepo.syncFolderPath.first()`, throws `FolderUnavailable` if null, returns `File(path, filename)`
    - `readFile`: resolves file, returns empty string if not exists, else `file.readText(Charsets.UTF_8)`. Catches `SecurityException` and throws `PermissionDenied`. Wraps in `withContext(ioDispatcher)`.
    - `writeFile`: resolves file, uses `android.util.AtomicFile` for safe writes. `startWrite()` -> `stream.write(content.toByteArray(Charsets.UTF_8))` -> `finishWrite(stream)`. On `IOException`, calls `failWrite(stream)` then rethrows. Wraps in `withContext(ioDispatcher)`.
    - `fileExists`: resolves file, returns `file.exists()`. Wraps in `withContext(ioDispatcher)`.
    - `listOrgFiles`: reads folder path, lists files filtering to `.org` extension, excluding `.sync-conflict` files (using `name.contains(".sync-conflict")`), excluding hidden files (starting with `.`). Returns filenames only. Wraps in `withContext(ioDispatcher)`. Returns `emptyList()` if path is null or folder doesn't exist.
    - Before every file operation, check `Environment.isExternalStorageManager()`. If false, throw `PermissionDenied("All files access permission not granted")`.

    **`AppPreferencesRepository.kt`:**
    - Top-level `private val Context.appDataStore: DataStore<Preferences> by preferencesDataStore(name = "sakura_prefs")`
    - Companion: `SYNC_FOLDER_PATH = stringPreferencesKey("sync_folder_path")`, `ONBOARDING_COMPLETE = booleanPreferencesKey("onboarding_complete")`
    - `syncFolderPath: Flow<String?>` -- reads from DataStore, catches IOException and emits `emptyPreferences()`
    - `onboardingComplete: Flow<Boolean>` -- reads from DataStore, defaults to false, catches IOException
    - `suspend fun setSyncFolderPath(path: String)` -- `dataStore.edit { it[SYNC_FOLDER_PATH] = path }`
    - `suspend fun setOnboardingComplete()` -- `dataStore.edit { it[ONBOARDING_COMPLETE] = true }`

    **`AppContainer.kt`** (update from stub):
    - `val prefsRepo = AppPreferencesRepository(context)`
    - `val syncBackend: SyncBackend = SyncthingFileBackend(prefsRepo)`
    - `val orgParser = OrgParser` (object reference)
    - `val orgWriter = OrgWriter` (object reference)

    **ANTI-PATTERNS to avoid (from research):**
    - Do NOT use `ContextCompat.checkSelfPermission` for MANAGE_EXTERNAL_STORAGE -- use `Environment.isExternalStorageManager()`
    - Do NOT call `dataStore.data.first()` in constructor -- always in a coroutine
    - Do NOT skip charset specification -- always `Charsets.UTF_8`
    - Do NOT let `.sync-conflict` filenames propagate past SyncBackend -- filter at this layer
  </action>
