    Constructor: `FoodLogViewModel(foodRepo: FoodRepository, prefsRepo: AppPreferencesRepository, savedStateHandle: SavedStateHandle)`

    State:
    - `val uiState: StateFlow<FoodLogUiState>` — primary screen data

    - `val expandedMeals: StateFlow<Set<String>>` — which meal sections are expanded

    - `val lastAddedEntry: StateFlow<Pair<Long, String>?>` — (entryId, mealLabel) for undo

    Key functions:
    - `loadDay(date: LocalDate)` — sets Loading, calls foodRepo.loadDay + prefsRepo.macroTargets, emits Success
    - `navigateToDate(date: LocalDate)` — update selectedDate, call loadDay
    - `navigatePrevDay()` / `navigateNextDay()` — convenience
    - `toggleMealExpanded(mealLabel: String)` — toggle in expandedMeals set
    - `addEntry(mealLabel: String, entry: FoodEntry)` — call foodRepo.addEntry, store lastAdded for undo, reload day
    - `updateEntry(mealLabel: String, entryId: Long, updated: FoodEntry)` — call foodRepo.updateEntry, reload day
    - `deleteEntry(mealLabel: String, entryId: Long)` — call foodRepo.deleteEntry, reload day
    - `undoLastAdd()` — use lastAddedEntry to call deleteEntry, clear lastAdded, reload day
    - `toggleEditMode()` — flips isEditMode (only applicable to past days)
    - `saveToLibrary(entry: FoodEntry)` — convert to FoodLibraryItem (generate UUID), call foodRepo.saveToLibrary
    - `saveMealAsTemplate(mealLabel: String, templateName: String)` — gather entries from current meal, create MealTemplate, call foodRepo.saveTemplate
    - `applyTemplate(mealLabel: String, template: MealTemplate)` — call foodRepo.applyTemplate, reload day
