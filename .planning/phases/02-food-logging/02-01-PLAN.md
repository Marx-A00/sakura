    **OrgFoodRepository.kt** — implements FoodRepository backed by org files via SyncBackend.

    Constructor takes: `syncBackend: SyncBackend`, `ioDispatcher: CoroutineDispatcher = Dispatchers.IO`

    **Critical:** All file mutations MUST be inside `fileMutex.withLock { ... }` (see RESEARCH Pitfall 3). Use a single Mutex instance per repository. Read operations do NOT need the mutex (reads are inherently safe; worst case is a slightly stale read which is acceptable for a single-user app).

    Implement each method:

    `loadDay(date: LocalDate)`:
    - Read food-log.org via syncBackend. If file doesn't exist (SyncBackendError.NotFound), return empty list.
    - Parse with `OrgParser.parse(content, ParseMode.FOOD)`
    - Find the OrgDateSection matching `date`
    - Convert each OrgMealGroup + OrgFoodEntry to domain MealGroup + FoodEntry
    - Return the list (may be empty if no section for that date)

    `addEntry(date: LocalDate, mealLabel: String, entry: FoodEntry)`:
    - fileMutex.withLock + withContext(ioDispatcher):
    - Read food-log.org (or empty string if not found)
    - Parse to OrgFile
    - Find or create OrgDateSection for date
    - Find or create OrgMealGroup for mealLabel within that section
    - Convert FoodEntry to OrgFoodEntry (with id = entry.id, blank name -> "Unnamed")
    - Add to entries list
    - Rebuild OrgFile with updated section
    - Write back via syncBackend.writeFile("food-log.org", OrgWriter.write(updatedFile))
    - The "find or create" logic: if no section for date exists, create one and insert it at the correct position (sorted by date descending). If no meal group for mealLabel exists within the section, create it.

    `updateEntry(date, mealLabel, entryId, updated)`:
    - fileMutex.withLock + withContext(ioDispatcher)
    - Read, parse, find section, find meal group, find entry by matching `:id:` property
    - Replace the entry
    - Write back

    `deleteEntry(date, mealLabel, entryId)`:
    - fileMutex.withLock + withContext(ioDispatcher)
    - Read, parse, find section, find meal group, remove entry by id
    - If meal group becomes empty after removal, remove the meal group
    - If date section becomes empty (no meals), remove the section
    - Write back

    `loadLibrary()`:
    - Read food-library.org. If not found, return empty list.
    - Parse with `OrgParser.parseLibrary(content)`
    - Convert OrgFoodEntry items to FoodLibraryItem domain objects

    `saveToLibrary(item: FoodLibraryItem)`:
    - fileMutex.withLock + withContext(ioDispatcher)
    - Read food-library.org (or create with "* Food Library\n" header if not found)
    - Parse, add item, write back

    `deleteFromLibrary(itemId: String)`:
    - fileMutex.withLock + withContext(ioDispatcher)
    - Read, parse, remove item by id, write back

    `loadTemplates()`:
    - Read meal-templates.org. If not found, return empty list.
    - Parse with `OrgParser.parseTemplates(content)`
    - Convert to domain MealTemplate objects

    `saveTemplate(template: MealTemplate)`:
    - fileMutex.withLock + withContext(ioDispatcher)
    - Read meal-templates.org (or create with "* Meal Templates\n" header)
    - Parse, add template, write back

    `applyTemplate(date, mealLabel, template)`:
    - For each FoodLibraryItem in template.entries: create a FoodEntry with a new id (System.currentTimeMillis() + index offset to ensure uniqueness), call addEntry for each. Because addEntry already acquires the mutex, applyTemplate should acquire the mutex ONCE for the entire batch — refactor to use an internal `addEntryInternal` that assumes the mutex is already held.

    `deleteTemplate(templateId: String)`:
    - fileMutex.withLock + withContext(ioDispatcher)
    - Read, parse, remove, write back

    `loadRecentItems(limit: Int)`:
    - Read food-log.org. Parse all sections.
    - Walk entries across all dates, collect distinct food names (most recent first)
    - Convert the last `limit` distinct entries to FoodLibraryItem format
    - This is a read-only operation so no mutex needed

    **Helper: Conversion functions**
    - `OrgFoodEntry.toFoodEntry()`: maps org model to domain model
    - `FoodEntry.toOrgFoodEntry()`: maps domain to org, substituting "Unnamed" for blank names
    - `OrgFoodEntry.toFoodLibraryItem(id: String)`: for library parsing
    - `FoodLibraryItem.toOrgFoodEntry()`: for library writing

    **Error handling:** Wrap all SyncBackend calls in try/catch. Return `Result.failure(e)` for I/O errors. `loadDay` and other read methods should catch `SyncBackendError.NotFound` and return empty results gracefully.
